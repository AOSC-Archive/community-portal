补丁包1.0.5
包名: cp-update
版本: 1.0.5
特性：
增加白大在http://forum.anthonos.org/thread-284-1-1.html发现的文件系统工具的缺失，增加了reiserfsprogs reiser4progs xfsprogs ntfsprogs jfsutils zfs-fuse
（是的，你没看错，ZFS）使用参照：http://www.ibm.com/developerworks/cn/linux/l-zfs/和https://wiki.archlinux.org/index.php/ZFS
为方便网络配置，增加Wicd以及相应的curses和gtk客户端
wicd-curses使用方法$ wicd-curseswicd-gtk须在图形界面使用，startx之后鼠标右键，参照图示


1.0.0版特性
这是根据白大在http://forum.anthonos.org/thread-273-1-1.html提的建议和发现的bug构建的修正包
修正root的桌面配置文件和python-dbus缺失的问题（一个很无奈的方法）
去除/etc/default/grub的quiet参数
增加桌面菜单和浏览器的汉化（同样无奈的方法）
在桌面菜单增加完整的Debian应用程序菜单（无奈的方法）
增加一个“运行”对话框（同上）

使用说明$ sudo -s
# axel http://anthonmirror.b0.upaiyun.com/centralpoint/aurora/cp-update_1.0.5~vanessa-1_all.deb
# dpkg -i cp-update_1.0.0~vanessa-1_all.deb
# apt-get -f install无奈的方法就是把新增的包写进cp-update的依赖里面
基本上只要是CP安装好dpkg都会报依赖缺失，所以用apt-get -f install来解决
安装时会提示是否将用户tux加入netdev组，请勾选，否则无法正常使用wicd
注意：当开机输出不再滚动时请按下回车键，登录提示符就会出现

Tengine 1.4.4 for CentralPoint
简介
Tengine是由淘宝网发起的Web服务器项目。它在Nginx的基础上，针对大访问量网站的需求，添加了很多高级功能和特性。Tengine的性能和稳定性已经在大型的网站如淘宝网，天猫商城等得到了很好的检验。它的最终目标是打造一个高效、稳定、安全、易用的Web平台。

从2011年12月开始，Tengine成为一个开源项目，Tengine团队在积极地开发和维护着它。Tengine团队的核心成员来自于淘宝、搜狗等互联网企业。Tengine是社区合作的成果，我们欢迎大家参与其中，贡献自己的力量。


特性
继承Nginx-1.2.7的所有特性，100%兼容Nginx的配置；
动态模块加载（DSO）支持。加入一个模块不再需要重新编译整个Tengine；
输入过滤器机制支持。通过使用这种机制Web应用防火墙的编写更为方便；
动态脚本语言Lua支持。扩展功能非常高效简单；
支持管道（pipe）和syslog（本地和远端）形式的日志以及日志抽样；
组合多个CSS、JavaScript文件的访问请求变成一个请求；
可以对后端的服务器进行主动健康检查，根据服务器状态自动上线下线；
自动根据CPU数目设置进程个数和绑定CPU亲缘性；
监控系统的负载和资源占用从而对系统进行保护；
显示对运维人员更友好的出错信息，便于定位出错机器；
更强大的防攻击（访问速度限制）模块；
更方便的命令行参数，如列出编译的模块列表、支持的指令等；
可以根据访问文件类型设置过期时间；
...
——摘录自http://tengine.taobao.org/

使用方法：$ sudo -s
# axel http://anthonmirror.b0.upaiyun.com/centralpoint/aurora/tengine_1.4.4~vanessa-1_amd64.deb
# dpkg -i tengine_1.4.4~vanessa-1_amd64.deb
# apt-get -f install注意：
该包的包名为nginx，使用前请卸载已安装的nginx
如果有Apache在运行，请注意不Tengine服务器会启动失败因为80端口被占用，请自行修改其中一个的端口
apt-get -f install为补齐依赖，必须
由于有端口占用的问题，请手动开启tengine的服务以及在需要时手工设置开机启动（服务: nginx）
